
Comandos básicos do git
	git init - dentro do diretório selecionado, inicia o monitoramento pelo git.
	git status - verifica a situação dos diretórios controlados. 
	git log - traz o log de todos os commits efetuados no diretório.

	rm -rf .git - remove o monitoramento do git, apagando o diretório de controle .git do diretório selecionado.


03 estágios do git

	Commited
		Gravado
		Significa que foi armazenado localmente com segurança
	Modified
		Modificado
		Arquivo foi alterado mas ainda não foi dado o "commit"
	Staged
		Selecionado
		Significa que marcou um arquivo modificado na versão corrente para ser parde do próximo commit

Workflow básico:
	Modifique os arquivos
	Selecione os arquivos que farão parte do próximo commit
	Faça o commit que utilizarão os arquivos "staged"


Andando entre commits/snapshots

	git checkout <commit id>
	git checkout master - retorna para o último commit.

Conhecendo o GitHub
	
	Reunião de repositórios de qualquer pessoa, de modo público. Pagando, pode ter repositórios privados.
	BitBucket permite ter repositórios privados de graça.

Clonando projetos
	git clone link(url do projeto).git
	Baixa o projeto (se for público) pra sua máquina e permite navegar entre os commits.
	Bom recurso para estudar projetos de outros, para aprendizado.

Criando e enviando projeto para o GitHub

	ls -la - verifica se o diretório tem repositório
	git remote add origin https://github.com/<usuário>/<repositório>.git - cria o link no git do diretório local para o diretório do GitHub.
	git push -u origin master - sobe os arquivos para o diretório linkado no git.

	
	git push - envia arquivos locais para o repositório selecionado. Autenticado com usuário e token gerado na plataforma GitHub (token válido por tempo determinado)
